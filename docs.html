⚽ World Cup Predictions

A React + Vite app that calculates and compares World Cup prediction scores across multiple scoring systems. Track your friends' predictions, see who's on top, and visualize performance over time.

Scoring Systems

The app ships with six scoring systems. Each takes the same input — a list of games and predictions — and produces a leaderboard. Switch between them using the dropdown at the top of the app.

Ted Classic Simple Max 4 pts/game

The original scoring system. Rewards predicting the correct outcome, goal difference, and exact score.

CategoryPointsCondition
Winner+2Correctly predicted home win, away win, or draw
Goal Difference+1Predicted goal difference (home − away) matches actual
Exact Score+1Both home and away scores match exactly
Example: Actual: Mexico 2–1 South Africa. You predicted 2–1. → Winner ✓ (+2), GD ✓ (+1), Exact ✓ (+1) = 4 points

Ted+ Simple Max 7 pts/game

Enhanced version of Ted Classic with additional categories and higher rewards for harder predictions.

CategoryPointsCondition
Winner+2 / +3+2 for correct home/away win, +3 for correctly predicting a draw (draws are harder)
Goal Difference+1Predicted goal difference matches actual
Total Goals+1Predicted total goals (home + away) matches actual
Exact Score+2Both scores match exactly (worth double vs. Ted Classic)
Example: Actual: 1–1 draw. You predicted 1–1. → Draw ✓ (+3), GD ✓ (+1), Total goals ✓ (+1), Exact ✓ (+2) = 7 points

Gambler's Context-Aware Variable pts

High risk, high reward. The fewer people who predicted the same outcome as you, the bigger your payoff when right — but the bigger your loss when wrong.

multiplier = totalPlayers / playersWhoPickedSameOutcome
base = 1 (correct winner) + 2 (if exact score)
Correct: total = round(base × multiplier)
Wrong: total = −round(multiplier)
CategoryPointsCondition
Base1–3+1 for correct winner, +2 bonus for exact score
Uniqueness BonusvariesCorrect: round(base × multiplier) − base
Risk Penalty−multiplierWrong: −round(multiplier) — contrarian misses hurt more
Win example: 11 players, only 2 predicted a draw, draw happens. multiplier = 5.5 → total = round(1 × 5.5) = +6 points.
Loss example: Same setup but home wins. Those 2 draw pickers: penalty = −round(5.5) = −6 points.

Ladder (ELO) Stateful

Competitive ranking using the ELO rating system from chess. Everyone starts at 1000. Each game, every pair of players is compared head-to-head — the better prediction wins the matchup.

Expected: EA = 1 / (1 + 10(RB − RA) / 400)
Rating change: Δ = k × (S − E), where k = 32 / (numPlayers − 1)
"Better prediction" is determined by Ted Classic raw score (0–4).
CategoryValueMeaning
Δ (Delta)+/− numberRating change for that game
TotalELO ratingCurrent rating (NOT a cumulative point sum)
Key insight: Beating a higher-rated player earns more ELO. Losing to a lower-rated player costs more. Equal predictions = draw (Δ ≈ 0).

Black Sheep Context-Aware Variable pts

Your benchmark is the crowd. Beat the average prediction and gain points; do worse than the crowd and lose points. Consistency matters.

crowdError = |avgHome − actualHome| + |avgAway − actualAway|
playerError = |predictedHome − actualHome| + |predictedAway − actualAway|
edge = crowdError − playerError
points = round(edge × 2) — positive or negative
bonus = +1 for exact score
SituationPointsExample
Beat crowd by 2+4Crowd error 3, your error 1 → edge 2 → +4 pts
Beat crowd by 0.5+1Crowd error 1.5, your error 1 → edge 0.5 → +1 pt
Tied with crowd0Same error as crowd → edge 0 → 0 pts
Worse than crowd−penaltyYour error > crowd error → negative points
Exact score+1 bonusAlways awarded on top of edge points
Key insight: Unlike Gambler's (outcome risk), Black Sheep tracks accuracy vs. the group mean. Consistently better predictions are rewarded; consistently worse ones are punished.

Hot Streak Stateful Max ~8 pts/game

Ted Classic base scoring plus escalating bonuses for consecutive correct winner predictions. Wrong winner resets the streak to zero.

CategoryPointsCondition
Base0–4Identical to Ted Classic scoring
Streak Bonus0, +1, +2, +3…+(n−1) for the nth consecutive correct winner
Example streak: Game 1: correct winner → streak = 1, bonus = +0 | Game 2: correct winner → streak = 2, bonus = +1 | Game 3: correct winner → streak = 3, bonus = +2 | Game 4: wrong winner → streak resets to 0

Equal Aggregate Meta Max 100 pts/game

A "consensus" ranking that combines all 6 base scoring systems equally. Each system's raw scores are normalized to a 0–100 scale per game using min-max normalization, then averaged.

normalized = (score − min) / (max − min) × 100
total = average of all 6 normalized scores
ColumnValueMeaning
TC0–100Normalized Ted Classic score
T+0–100Normalized Ted+ score
Gam0–100Normalized Gambler's score
BS0–100Normalized Black Sheep score
Lad0–100Normalized Ladder (ELO) delta
HS0–100Normalized Hot Streak score
Key insight: A player must perform well across all scoring philosophies to rank highly. Dominating one system but failing at others won't save you.

Weighted Aggregate Meta Max 100 pts/game

Same normalized approach as Equal Aggregate, but each system is scaled by a configurable weight before averaging. Emphasizes bolder and more competitive systems.

total = Σ(normalizedi × weighti) / Σ(weighti)
SystemWeightRationale
Ted Classic×1.0Baseline scoring
Ted+×1.0Enhanced baseline
Gambler's×2.0Rewards bold, contrarian predictions
Black Sheep×1.5Rewards individual insight over groupthink
Ladder (ELO)×1.5Competitive head-to-head ranking
Hot Streak×1.5Rewards consistency over time
Customization: Edit the weights array in src/utils/scoring/weightedAggregate.ts to change the emphasis. The total weight sum is calculated automatically.

System Type Badges

BadgeMeaningSystems
Simple Stateless per-game scoring — each prediction is scored independently Ted Classic, Ted+
Context-Aware Needs all predictions for a game to calculate (e.g., uniqueness multiplier) Gambler's, Black Sheep
Stateful Maintains cumulative state across sequential games (ELO, streaks) Ladder, Hot Streak
Meta Aggregates results from all base scoring systems into a unified ranking Equal Aggregate, Weighted Aggregate

Key Types

// A game (played or unplayed)
interface Game {
  id: number;           // Sequential — games are played in ID order
  home: string;         // Home team name
  away: string;         // Away team name
  homeScore: number | null;  // null = not yet played
  awayScore: number | null;
}

// A player's prediction for one game
interface Prediction {
  name: string;         // Player name
  gameId: number;       // References Game.id
  homeScore: number;
  awayScore: number;
}

// Points earned — flexible categories per scoring system
interface PointBreakdown {
  categories: Record<string, number>;  // e.g. { winner: 2, goalDifference: 1 }
  total: number;
}

// A scoring system definition
interface ScoringSystem {
  name: string;
  description: string;
  categoryLabels: { key: string; label: string }[];  // drives UI columns
  maxPerGame?: number;   // for color-scaling in the UI
  calculateStandings: (games: Game[], predictions: Prediction[]) => PlayerScore[];
}

How Scoring Systems Work Internally

Each scoring system is an object that conforms to the ScoringSystem interface. The key method is calculateStandings, which receives all games and predictions and returns sorted player standings.

There are three patterns:

PatternHelperUsed By
Simple — stateless per-game simpleScoring(fn) wraps a (game, prediction) => PointBreakdown function Ted Classic, Ted+
Context-aware — needs all predictions per game Custom calculateStandings method Gambler's, Black Sheep
Stateful — cumulative state across games Custom calculateStandings method with state maps Ladder, Hot Streak

How-To Guides

Add a New Game

  1. 1
    Open src/data/games.csv and add a new row with the next sequential ID, team names, and empty scores: 
    3;Germany;Japan;;
  2. 2
    Add each player's prediction for the new game in src/data/predictions.csv: 
    Raf;3;2;0
Didi;3;1;1

Record a Game Result

  1. 1
    Open src/data/games.csv and fill in the home_score and away_score for the game: 
    3;Germany;Japan;2;1
    The leaderboard, chart, and all scoring systems will update automatically on save (via Vite HMR).

Add a New Player

  1. 1
    Add prediction rows for the new player in src/data/predictions.csv. Use the same name value across all their predictions: 
    NewPlayer;1;2;0
NewPlayer;2;1;1
    The player will appear in the leaderboard automatically.

Add a New Scoring System

  1. 1
    Create a new file in src/utils/scoring/, e.g., mySystem.ts.
  2. 2
    Define your system as a ScoringSystem object with a default export: 
    import type { ScoringSystem } from '../../types';
import { simpleScoring } from './helpers';  // if stateless

const mySystem: ScoringSystem = {
  name: 'My System',
  description: 'Short description of the rules',
  categoryLabels: [
    { key: 'myCategory', label: 'My Cat' },
  ],
  maxPerGame: 5,  // for UI color scaling

  // Option A: Simple (stateless per-game)
  calculateStandings: simpleScoring((game, prediction) => {
    // Return { categories: { myCategory: points }, total: points }
  }),

  // Option B: Custom (context-aware or stateful)
  // calculateStandings(games, predictions) { ... }
};

export default mySystem;
  3. 3
    Register it in src/utils/scoring/index.ts: 
    import mySystem from './mySystem';

export const scoringSystems: ScoringSystem[] = [
  tedClassic,
  tedPlus,
  // ...
  mySystem,  // ← add here
];
    The new system will appear in the dropdown automatically.

Available Helpers

When building a new scoring system, these shared utilities from helpers.ts are available:

FunctionReturnsDescription
getWinner(homeScore, awayScore) 'home' | 'away' | 'draw' Determines the match outcome
simpleScoring(scoreFn) calculateStandings Wraps a per-game function into a full standings pipeline (handles grouping, sorting)
tedClassicRawScore(game, prediction) number (0–4) Ted Classic score — useful as a base measure in composite systems

Key Design Decisions

  • PointBreakdown.categories uses Record<string, number> — each system defines its own keys via categoryLabels, keeping the UI fully dynamic.
  • Each ScoringSystem owns its calculateStandings — no forced shared pipeline. The simpleScoring helper is opt-in for stateless systems.
  • Ladder's totalPoints = final ELO rating (not a cumulative sum). The chart component detects this and adjusts the Y-axis accordingly.
  • CSV files are imported as raw strings via Vite's ?raw suffix, so they're bundled at build time and benefit from HMR during development.
  • Games are always processed in sequential ID order, which is critical for stateful systems (Ladder, Hot Streak).

Modifying the UI

  • Leaderboard columns — adapt automatically from system.categoryLabels. No changes needed when adding new scoring systems.
  • Game detail columns — same dynamic rendering via categoryLabels in GameCard.tsx.
  • Point color scaling — uses system.maxPerGame to set thresholds (perfect ≥ max, great ≥ 75%, good ≥ 50%, ok ≥ 1, zero = 0).
  • Chart — shows cumulative scores (or ELO rating) over time. Colors are from a fixed 12-color palette.
  • All styles — in src/App.css and src/index.css. Uses CSS custom properties for dark/light mode.
